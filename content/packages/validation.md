# `@orlo/validation`

Use `@orlo/validation` when you need deterministic output checks without adopting the full platform.

## Includes

- schema validation
- rule-based validation
- validation summarization
- custom-rule sandboxing with Node `vm`

## Best for

- response validation
- model output quality gates
- reproducible guardrail logic
